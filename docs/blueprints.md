# 架構藍圖索引 (Architecture Blueprints)

本文件是專案架構文檔的頂層索引，旨在提供一個快速訪問所有核心設計規範的入口。

## 核心架構文件

1.  **[專案架構 (Project Architecture)](./architecture.md)**
    *   **內容**: 描述了以「功能切片」為核心的高階架構模型，以及三大邏輯平面的劃分。
    *   **用途**: 了解專案的整體設計哲學。

2.  **[架構邊界 (Architectural Boundaries)](./boundaries.md)**
    *   **內容**: 定義了功能切片之間以及切片內部的嚴格依賴規則。
    *   **用途**: 開發時必須遵守的程式碼引用規則，防止不合理的耦合。

3.  **[專案互動模型 (Interaction Model)](./interactions.md)**
    *   **內容**: 透過圖表展示了不同架構層和功能切片之間的數據流與控制流。
    *   **用途**: 視覺化地理解系統各部分是如何協同工作的。

4.  **[程式碼規範 (Coding Conventions)](./conventions.md)**
    *   **內容**: 詳細說明了目錄、檔案和變數的命名約定。
    *   **用途**: 確保整個專案的程式碼風格保持一致。

5.  **[術語詞彙表 (Glossary)](./glossary.md)**
    *   **內容**: 解釋了專案中使用的特定架構術語的準確含義。
    *   **用途**: 統一團隊溝通語言，避免因術語理解不一致而導致的混淆。
